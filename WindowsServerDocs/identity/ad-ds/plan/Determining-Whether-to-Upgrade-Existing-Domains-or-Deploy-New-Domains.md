# <a name="determining-whether-to-upgrade-existing-domains-or-deploy-new-domains"></a>Per determinare se aggiornare domini esistenti o distribuire nuovi domini

>Si applica a: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Ogni dominio nella progettazione che sarà un nuovo dominio o un oggetto esistente aggiornati dominio. Gli utenti di domini esistenti che non devono essere spostati nei nuovi domini.  
  
Spostamento tra domini di account, gli utenti finali può influire. Prima di decidere se gli utenti di spostare in un nuovo dominio o eseguire l'aggiornamento di domini esistenti, valutare i vantaggi a lungo termine amministrativi di un nuovo dominio di Active Directory con i costi di spostamento degli utenti nel dominio.  
  
Per ulteriori informazioni sull'aggiornamento di domini di Active Directory per Windows Server 2008, vedere [l'aggiornamento di domini Active Directory per Windows Server 2008 e Windows Server 2008 R2 AD DS domini](https://technet.microsoft.com/library/cc731188.aspx).  
  
Per ulteriori informazioni su ristrutturazione dei domini di Active Directory all'interno e tra foreste, vedere Utilità di migrazione ad Active Directory versione 3.1 Guida alla migrazione ([https://go.microsoft.com/fwlink/?LinkId=93678](https://go.microsoft.com/fwlink/?LinkId=93678)).  
  
Per un foglio di lavoro agevolare la documentazione i piani per i domini nuovi e aggiornati, scaricare Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip).
